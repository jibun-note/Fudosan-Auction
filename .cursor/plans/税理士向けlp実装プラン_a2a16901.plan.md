# 税理士向け不動産オークション LP 実装プラン

## 技術スタック（現状のまま）

- **フレームワーク**: Next.js 16 (App Router)、React 19
- **スタイル**: Tailwind CSS v4（既存 [app/globals.css](app/globals.css)）
- **画像・ボタン**: プレースホルダー（ダミー）で対応。画像は `placeholder` 系 or 単色/グラデーションの div、ボタンは `#` または `javascript:void(0)` のリンク/button

---

## デザイン方針

- **カラー**: ネイビー（メイン背景・ヘッダー）、ゴールド/アンバー（CTA・アクセント・下線）、白（カード・テキストエリア）、薄い水色（紹介スキーム・税務ブロック）
- **フォント**: 既存の Geist（[app/layout.tsx](app/layout.tsx) で読み込み済み）。日本語表示を考慮し、必要なら `font-sans` で Noto Sans JP 等を追加検討
- **レイアウト**: セクションごとに `max-w-6xl` または `max-w-7xl` で中央寄せ、余白は `py-16`〜`py-24`、`px-6` で統一

---

## ファイル構成

```
app/
  layout.tsx          # 既存。metadata を LP 用に更新
  page.tsx             # 全セクションを並べるメインページ
  globals.css          # LP 用の CSS 変数（ネイビー・ゴールド等）を追加
components/
  Header.tsx           # ロゴ + ナビ + 「無料で資料請求」
  Hero.tsx             # ヒーロー見出し・サブ・2CTA + 背景ダミー
  SchemeSection.tsx    # 税理士に"価値"が返る売却スキーム（3カード + 紹介スキーム例）
  MaximizeSection.tsx  # オークションが顧問先の利益を最大化（画像ダミー + 3利点）
  ComparisonTable.tsx  # 仲介/買取/オークション比較表
  CaseStudySection.tsx # 実例でわかる差額と税務効果（事例・棒グラフ風・CTA）
  FlowSection.tsx      # ご利用の流れ（5ステップ）
  FaqSection.tsx       # よくあるご質問（アコーディオン・クライアントコンポーネント）
  Footer.tsx           # フッターCTA + 会社情報 + サイトマップ + お問い合わせ
```

- **画像**: `public/` に `hero-placeholder.svg` や `maximize-placeholder.svg` を置くか、あるいは `components` 内で `div` で代替（例: 背景 `bg-neutral-200` + 縦横比 `aspect-video`）でダミー表示
- **ボタン**: すべて `<a href="#">` または `<button type="button">` でダミー。スタイルのみ実装
- **アイコン**: **react-icons** を利用。`npm install react-icons` で追加し、各セクションで以下からインポートして使用する。

## セクション別実装内容

### 1. グローバルスタイル・メタデータ

- **globals.css**: `:root` に LP 用変数を追加（例: `--navy`, `--gold`, `--accent-blue`）。既存の `--background` / `--foreground` はそのままでも可
- **layout.tsx**: `metadata.title` / `metadata.description` を税理士向け LP の文言に変更

### 2. Header

- 左: サービス名「不動産オークション」（テキストまたはロゴ用ダミー）
- 中央〜右: ナビリンク（メリット・他方式との比較・成功事例・利用の流れ・FAQ・会社情報）。各項目は `#メリット` 等のアンカーリンクで同一ページ内スクロール
- 右端: CTA「無料で資料請求」（ゴールド系ボタン、`href="#"`）

### 3. Hero

- 背景: ネイビー。画像はダミー（`div` でグラデーション or `public` の1枚）
- 見出し: 「税理士の顧問先の不動産、もっと高く・安心して売却」
- サブ: 「不動産オークション方式で、顧問先の資産価値を最大化。」
- ボタン2つ: 「無料で資料請求する」「顧問先向け説明資料DL」（いずれも `href="#"`）

### 4. 税理士に"価値"が返る売却スキーム（SchemeSection）

- 見出し + ゴールドのアンダーライン
- 3カード（横並び・レスポンシブで縦積み）: 顧客満足度アップ / 信頼強化 / 差別化。各カードは **react-icons** でアイコン表示:
    - 顧客満足度アップ: `FaThumbsUp`（`react-icons/fa`）
    - 信頼強化: `HiOutlineShieldCheck`（`react-icons/hi2`）
    - 差別化: `HiOutlineUserGroup`（`react-icons/hi2`）
- 下段: 水色背景の「紹介スキーム例」ブロック。説明文 + 「紹介スキーム資料はこちら≫」（`href="#"`）

### 5. オークションが顧問先の利益を最大化（MaximizeSection）

- 見出し + アンダーライン
- 2カラム: 左に画像ダミー（`aspect-video` の div 等）、右に番号付き3利点。各項目は **react-icons** の丸数字風アイコン + タイトル + 説明文:
    -   1. 高額売却の可能性: `HiOutlineBanknotes`（`react-icons/hi2`）
    -   1. 透明性・安心感: `HiOutlineEye`（`react-icons/hi2`）
    -   1. 税務シミュレーション連動: `HiOutlineCalculator`（`react-icons/hi2`）

### 6. 仲介/買取/オークションの比較（ComparisonTable）

- 見出し: 「仲介/買取/オークションの違いが一目で」
- 表: 行＝売却価格・スピード・透明性・顧客満足度、列＝一般仲介・不動産買取・オークション。オークション列は背景色で強調
- 注釈: 「※価格保証/最低落札価格の設計も可能…」
- 下: 「より詳細な比較資料をダウンロード」（`href="#"`）

### 7. 実例でわかる差額と税務効果（CaseStudySection）

- 見出し: 「実例でわかる"差額"と"税務効果"」
- 事例見出し: 【匿名事例】築20年区分マンション(都心)
- 査定額 6,800万円 vs 落札額 7,320万円（+520万円 UP）を数字 + 棒グラフ風（Tailwind の `w-` で長さ表現）で表示
- 税務ブロック: 薄い水色背景。譲渡所得の式・注釈
- CTA ボタン: 「匿名事例集・税務シミュレーション相談」（`href="#"`）

### 8. ご利用の流れ（FlowSection）

- 見出し: 「ご利用の流れ」
- 5ステップを横並び（中央で線で接続）。各ステップ: **react-icons** でアイコン + タイトル + 短文説明。レスポンシブでは縦並びに変更。
    -   1. 資料請求・査定: `HiOutlineDocumentText`（`react-icons/hi2`）
    -   1. 最低落札価格設定: `HiOutlineCurrencyYen`（`react-icons/hi2`）
    -   1. オークション開催: `HiOutlineGavel`（`react-icons/hi2`）
    -   1. 成約・決済: `HiOutlineHandshake`（`react-icons/hi2`）
    -   1. アフターフォロー: `HiOutlinePhone` または `HiOutlineChatBubbleBottomCenterText`（`react-icons/hi2`）

### 9. よくあるご質問（FaqSection）

- 見出し: 「よくあるご質問」
- アコーディオン: 1つ目「一般仲介との一番の違いは何ですか？」は初期表示で開く。他は「もし売れなかった場合は？」「手数料・コストについて」「顧問税理士としてどこまで関与できますか？」は閉じた状態
- 開閉インジケーター: **react-icons** の `HiOutlineChevronDown`（`react-icons/hi2`）を回転で開閉表現
- **クライアントコンポーネント**: `'use client'` + `useState` で開閉。各項目はボタンで開閉、回答は条件付き表示

### 10. Footer

- 上段: ネイビー背景。「顧問先の資産価値を最大化する新しい一手を、今すぐご提案ください。」+ 補足 + 「無料で資料請求する」「オンライン説明会に申込む」（`href="#"`）
- 下段: 3カラム（会社情報・サイトマップ・お問い合わせ）。会社名・住所・免許・加盟団体・電話・メールはダミーテキストで記載
- 最下部: 「プライバシーポリシー｜利用規約」、© 表記

## 実装順序の提案

1. **react-icons** をインストール（`npm install react-icons`）
2. **globals.css** に LP 用カラー変数を追加
3. **Header** と **Footer** を実装し、[app/page.tsx](app/page.tsx) でレイアウトの骨組みを確認
4. **Hero** → **SchemeSection**（react-icons）→ **MaximizeSection**（react-icons）→ **ComparisonTable** → **CaseStudySection** → **FlowSection**（react-icons）→ **FaqSection**（react-icons の ChevronDown）の順でコンポーネントを追加し、page に並べる
5. 各セクションに `id` を付与し、Header のナビからアンカーリンクで飛べるようにする
6. **layout.tsx** の metadata を LP 用に更新
7. スマホ表示でナビ・表・5ステップが読みやすいよう、Tailwind の `md:` / `lg:` でレスポンシブを調整

---

## 補足

- **アイコン**: 上記のとおり **react-icons**（Font Awesome 5 + Heroicons 2）からインポートして使用。サイズ・色は Tailwind の `className`（例: `text-2xl text-amber-600`）で統一
- **FAQ の開閉**: サーバーコンポーネントの親から「開いている index」だけ props で渡し、FaqSection 内で state 管理する形でも可
- **画像ダミー**: `next/image` は使わず、`<div className="bg-neutral-300 aspect-video ..." />` や `placeholder.com` の URL を一枚だけ使う方法も可。いずれも「一旦ダミー」の範囲で対応
